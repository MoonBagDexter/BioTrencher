<task type="auto">
  <name>Task 2: Signal detector and rug filter pipeline stages</name>
  <files>src/pipeline/signal-detector.ts, src/pipeline/rug-filter.ts</files>
  <action>
    1. Create src/pipeline/signal-detector.ts:
       - Import bus, Signal type, logger, bs58
       - Constants:
         PUMP_PROGRAM = '6EF8rrecthR5Dkzon8Nwu78hRvfCKubJ14M5uBEwF6P'
         BUY_DISCRIMINATOR = Buffer.from('66063d1201daebea', 'hex')
         SELL_DISCRIMINATOR = Buffer.from('33e685a4017f83ad', 'hex')
         Account indices: MINT=2, USER=6
       - Export function initSignalDetector(): void
         - Subscribe to bus.on('webhook:received', handler)
         - Handler decodes each instruction in the transaction:
           a. Check if programIdIndex points to PUMP_PROGRAM in accountKeys
           b. Decode instruction data from base58 using bs58
           c. Compare first 8 bytes against BUY/SELL discriminators
           d. Extract mint and user from accountKeys using the account indices array
           e. If valid PumpFun buy/sell, emit bus.emit('signal:detected', signal)
           f. If no PumpFun instruction found, silently skip (not all txs are PumpFun)
         - Wrap in try/catch -- log errors but never crash. Malformed webhooks should be logged and skipped.
         - Log each detected signal at info level: type, mint (first 8 chars), user (first 8 chars), signature (first 12 chars)

    2. Create src/pipeline/rug-filter.ts:
       - Import bus, Signal, ValidatedSignal, logger, Connection from @solana/web3.js, config
       - Export function initRugFilter(connection: Connection): void
         - Subscribe to bus.on('signal:detected', handler)
         - Handler (async):
           a. Only process BUY signals (SELL signals should be emitted directly as they bypass rug check -- emit signal:detected sells forward differently, see note below)
           b. For BUY: call connection.getTokenLargestAccounts(new PublicKey(signal.mint)) and connection.getTokenSupply(new PublicKey(signal.mint)) in parallel via Promise.all
           c. Calculate each holder's percentage of total supply
           d. If ANY holder has >5%: emit bus.emit('signal:rejected', signal, reason) with reason like "Wallet holds X.X% of supply (limit: 5%)"
           e. If all holders <=5%: emit bus.emit('signal:validated', { ...signal, rugCheckPassed: true })
           f. Log result at info level: passed/rejected with reason
         - Handle RPC errors gracefully -- if getTokenLargestAccounts fails, reject the signal with reason "RPC error during rug check" (fail closed, not open)

       NOTE on SELL signals: SELL signals don't go through rug filter. They represent tracked wallets selling, which triggers copy-sell exit logic. The signal detector should still emit them as signal:detected, but the rug filter should forward SELL signals directly by emitting a separate event or by having the position manager listen to signal:detected for SELL type. Cleanest approach: rug filter ignores SELL signals entirely, and position manager subscribes to signal:detected directly for SELL-type signals.
  </action>

  </verify>
