<task type="auto">
  <name>Task 2: Core types, event bus, config, logger, and persistence</name>
  <files>src/core/types.ts, src/core/events.ts, src/server/config.ts, src/utils/logger.ts, src/utils/persistence.ts, data/wallets.json</files>
  <action>
    1. Create src/core/types.ts with all shared types:
       - RawWebhookPayload: { signature: string; transaction: { message: { accountKeys: string[]; instructions: { programIdIndex: number; accounts: number[]; data: string }[] } } } (array of these from Helius)
       - Signal: { type: 'BUY' | 'SELL'; mint: string; user: string; signature: string; timestamp: number }
       - ValidatedSignal: Signal & { rugCheckPassed: true }
       - ExitReason: 'copy-sell' | 'take-profit' | 'stop-loss' | 'time-limit'
       - ExitParams: { takeProfitPct: number; stopLossPct: number; timeLimitMs: number }
       - TradeResult: { signature: string; mint: string; solAmount: number; tokensReceived: number; entryPrice?: number; exitPrice?: number; mode: 'SAFE' | 'LIVE'; reason?: ExitReason; timestamp: number }
       - Position: { id: string; mint: string; entryPrice: number; tokensHeld: number; solInvested: number; currentPrice: number; pnlPct: number; exitParams: ExitParams; openedAt: number; status: 'open' | 'closed'; copiedFrom: string } (copiedFrom = wallet address that triggered the buy)
       - Config: { heliusApiKey: string; rpcUrl: string; jupiterApiKey: string; webhookBaseUrl: string; port: number; mode: 'SAFE' | 'LIVE'; wallets: string[]; maxPositions: number; buyAmountSol: number }

    2. Create src/core/events.ts:
       - Import TypedEmitter from tiny-typed-emitter
       - Define PipelineEvents interface with these events:
         'webhook:received': (payload: RawWebhookPayload) => void
         'signal:detected': (signal: Signal) => void
         'signal:validated': (signal: ValidatedSignal) => void
         'signal:rejected': (signal: Signal, reason: string) => void
         'signal:skipped': (signal: Signal, reason: string) => void
         'trade:executed': (result: TradeResult) => void
         'position:opened': (position: Position) => void
         'position:updated': (position: Position) => void
         'position:closed': (position: Position, reason: ExitReason) => void
         'price:updated': (prices: Map<string, number>) => void
       - Export a singleton `bus` instance with setMaxListeners(20)

    3. Create src/utils/logger.ts:
       - Create and export a pino logger instance
       - Set level from LOG_LEVEL env var or default 'info'
       - Use pino-pretty transport in development (optional -- only if installed)

    4. Create src/utils/persistence.ts:
       - Export async function readJson<T>(filePath: string, defaultValue: T): Promise<T> -- reads and parses JSON, returns defaultValue if file doesn't exist
       - Export async function writeJson(filePath: string, data: unknown): Promise<void> -- uses write-file-atomic for crash-safe writes
       - Ensure data/ directory exists (use fs.mkdir with recursive: true)

    5. Create src/server/config.ts:
       - Export function loadConfig(): Config
       - Load from process.env using dotenv
       - Load wallets from data/wallets.json (or path from WALLETS_FILE env var)
       - Validate each wallet address is a valid base58 Solana public key (use @solana/web3.js PublicKey constructor -- if it throws, the address is invalid)
       - Fail fast with clear error if any required env var is missing or any wallet address is invalid
       - Default maxPositions: 5, buyAmountSol: 0.1, port: 3000, mode: 'SAFE'

    6. Create data/wallets.json with an example structure:
       ["EXAMPLE_WALLET_ADDRESS_1", "EXAMPLE_WALLET_ADDRESS_2"]
       Add a comment in config.ts noting these must be replaced with real addresses.

    All exports should be named exports (no default exports).
  </action>
