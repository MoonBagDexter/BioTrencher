<task type="auto">
  <name>Task 1: Price monitor with batched Jupiter API polling</name>
  <files>src/pipeline/price-monitor.ts</files>
  <action>
    Create src/pipeline/price-monitor.ts:

    1. Export function initPriceMonitor(positionManager: PositionManager, config: Config): void

    2. Core function fetchPrices(mints: string[]): Promise<Map<string, number>>
       - If mints is empty, return empty Map
       - Call Jupiter Price API v3: GET https://api.jup.ag/price/v3?ids={comma-separated-mints}
       - Include header: x-api-key from config.jupiterApiKey
       - Parse response: data[mint].price for each mint
       - On HTTP 429 (rate limit): log warning, return empty Map (positions keep last known price)
       - On any other error: log error, return empty Map
       - Use native fetch (available in Node 18+)

    3. Price polling loop:
       - Use setInterval at 10 second intervals (conservative to avoid rate limits)
       - Each tick:
         a. Get active mints from positionManager.getActiveMints()
         b. If no active mints, skip this tick
         c. Call fetchPrices(mints)
         d. If prices returned, emit bus.emit('price:updated', prices)
       - Log at debug level each poll cycle

    4. Time-based exit checking (separate from price):
       - Use a second setInterval at 30 second intervals
       - Each tick: call positionManager.checkTimeLimitExits()
       - This ensures time-based exits fire even during quiet price periods (Pitfall 5 from research)

    5. Exponential backoff on consecutive 429s:
       - Track consecutive rate limit hits
       - After 3 consecutive 429s, double the polling interval (10s -> 20s -> 40s, max 60s)
       - Reset to 10s after a successful fetch
       - Log when backing off and when recovering

    6. Wire the price getter into SafeExecutor:
       - Call setPriceGetter on the trade executor with a function that calls fetchPrices for a single mint
       - This gives SafeExecutor access to real prices for mock buys
  </action>
