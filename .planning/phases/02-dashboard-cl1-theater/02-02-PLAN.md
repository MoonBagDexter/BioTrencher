  <action>
    1. Create `public/js/neuron-grid.js`:
       - Expose `window.NeuronGrid` object with `init()` and `handleMessage(msg)` methods
       - Init: get canvas#neuron-canvas, set up 2D context, handle HiDPI (canvas.width = clientWidth * devicePixelRatio, ctx.scale(dpr, dpr))
       - Create 8x8 tile array: each tile has { col, row, baseColor, intensity (0-1), targetIntensity, type ('autonomous'|'stimulated') }
         - Base colors: autonomous tiles = teal (#00d4aa), stimulated = olive (#88aa44), a few pink/red (#ff4488) scattered
         - Assign types randomly at init (70% autonomous, 20% stimulated, 10% pink accent)
       - requestAnimationFrame loop (cap delta at 100ms to prevent tab-switch jump):
         - Lerp each tile.intensity toward tile.targetIntensity at rate 0.05
         - Apply random ambient drift: every ~2s, randomly bump 3-5 tiles' targetIntensity to 0.3-0.6 (background neural activity)
         - Draw each tile as filled rect with globalAlpha = 0.15 + intensity * 0.85
         - Draw small white electrode dot (2px radius circle) at center of each tile, alpha = intensity
         - Draw connection lines (thin, accent-teal, low alpha 0.15) between tiles with intensity > 0.5 that are within 2 tiles distance
       - handleMessage(msg): react to trading events:
         - signal:detected / signal:validated: burst pattern -- set 8-12 random tiles to targetIntensity 1.0 (teal for buys)
         - signal:rejected: flash 4-6 tiles to pink/red targetIntensity 1.0
         - trade:executed: wave pattern -- sequentially raise tiles from center outward
         - position:closed: brief all-dim then recovery (all tiles targetIntensity 0.1, then ambient resumes)
         - price:updated: subtle pulse on 2-3 random tiles

    2. Create `public/css/neuron-grid.css`:
       - #neuron-canvas: width 100%, height 100%, display block, background transparent
       - Add subtle glow effect on the panel border when grid is active (CSS transition on border-color)

    3. Create `public/js/bio-stats.js`:
       - Expose `window.BioStats` object
       - Internal state: { neurons: 64, coherence: 0.72, entropy: 0.45, firingRate: 0, winRate: 0 }
       - Track closedWins/closedTotal for real win rate
       - Track event timestamps array for firing rate (events in last 60s rolling window)
       - Track previous prices for entropy calculation (price volatility)
       - Methods:
         - `recordEvent()`: push Date.now() to timestamps, trim older than 60s, update firingRate
         - `updateCoherence(tradeWon)`: EMA smoothing (alpha=0.1), rises on wins, drops on losses
         - `updateEntropy(priceChangePcts)`: avg absolute price change normalized to 0-1 range
         - `updateWinRate(won)`: increment closedWins/closedTotal, calculate ratio
         - `getStats()`: returns current stats object
       - handleMessage(msg): route WS messages to stat updates:
         - position:closed: updateCoherence (check pnlPct > 0), updateWinRate
         - price:updated: calculate price change percentages, updateEntropy
         - Any event: recordEvent() for firing rate
       - Slow drift: setInterval every 2s, add tiny random walk to coherence (+/- 0.005) and entropy (+/- 0.008) to feel alive

    4. Create `public/js/stats.js`:
       - Expose `window.Stats` object with `init()` and `update()` methods
       - init(): cache DOM references to stat value elements (stat-neurons, stat-coherence, etc.)
       - update(): read BioStats.getStats(), update DOM text content:
         - neurons: always 64
         - coherence: format to 2 decimal places (e.g., "0.72")
         - entropy: format to 2 decimal places
         - firingRate: integer + " Hz"
         - winRate: percentage format (e.g., "67%")
       - Add CSS class pulse animation when values change significantly (> 5% change)
       - Call update() on a 1s interval AND on every BioStats change
       - Tooltip data-title attributes should already be in HTML from plan 01

    5. Update `public/index.html`:
       - Add link to css/neuron-grid.css in head
       - Add script tags before app.js (order: bio-stats.js, confidence.js, stats.js, neuron-grid.js, positions.js, boot.js, app.js)
       - Ensure stat elements in status bar have correct IDs matching stats.js expectations
